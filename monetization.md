#### **Step 4: Verify Webhook**

```typescript
import crypto from 'crypto';

function verifyWebhook(payload: string, signature: string, secret: string) {
  const hmac = crypto.createHmac('sha256', secret);
  const digest = hmac.update(payload).digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(digest)
  );
}
```

#### **Step 5: Process Order**

```typescript
async function handleOrderCreated(order: Order) {
  // 1. Save order to database
  await db.orders.create({
    lemonSqueezyId: order.id,
    customerId: order.attributes.customer_id,
    total: order.attributes.total,
    status: order.attributes.status,
  });

  // 2. Send confirmation email
  await sendOrderConfirmation(order);

  // 3. Grant product access
  await grantProductAccess(order);

  // 4. Generate license keys (if applicable)
  if (order.attributes.first_order_item.product_id === 'digital-product') {
    await generateLicenseKey(order);
  }
}
```

## üîê Security Considerations

1. **API Keys**
   - Never expose in client-side code
   - Use environment variables
   - Separate test/live keys

2. **Webhook Verification**
   - Always verify signatures
   - Use timing-safe comparison
   - Log failed verifications

3. **Product Access**
   - Verify purchase before granting access
   - Check license key validity
   - Implement rate limiting

4. **Payment Data**
   - Never store credit card info
   - Let Lemon Squeezy handle PCI compliance
   - Only store order metadata
